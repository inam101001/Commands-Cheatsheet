🏗️ Images — Build

Basic Build
docker build .Build image from Dockerfile in current directory
docker build -t myapp:1.0 .Build and tag image as myapp:1.0
docker build -t myapp:latest -t myapp:1.2 .Apply multiple tags in one build
docker build -f Dockerfile.prod .Use a specific Dockerfile path
docker build --no-cache .Build without using layer cache
docker build --pull .Always pull newer base image
docker build --target builder .Stop at a named stage (multi-stage)
docker build --build-arg KEY=val .Pass build-time variable (ARG)
docker build --build-arg NODE_ENV=prod --build-arg API=x .Multiple build args
docker build --label version=1.0 .Add label to built image
docker build --compress .Compress build context with gzip
docker build --squash .Squash all layers into one (experimental)
docker build --network=host .Use host network during build
docker build --shm-size 256m .Set /dev/shm size for build containers
docker build --memory 2g .Limit memory for build
docker build --cpu-quota 50000 .Limit CPU quota during build
docker build --add-host host.docker.internal:host-gateway .Add /etc/hosts entry for build
docker build --secret id=mysecret,src=./secret.txt .Mount build secret (BuildKit)
docker build --ssh default .Forward SSH agent to build (BuildKit)
docker build --progress=plain .Show full build output (no TTY collapsing)
docker build --iidfile ./image.id .Write image ID to file
DOCKER_BUILDKIT=1 docker build .Enable BuildKit explicitly
Context & Input
docker build - < DockerfileRead Dockerfile from stdin (no build context)
docker build https://github.com/user/repo.gitBuild from remote Git URL
docker build https://github.com/user/repo.git#branch:dirSpecific branch and subdirectory
docker build http://example.com/archive.tar.gzBuild from remote tar archive
cat archive.tar.gz | docker build -Build from stdin tar archive
Buildx (Extended Builder)
docker buildx create --name mybuilder --useCreate and activate new buildx builder
docker buildx create --driver docker-container --name mybuilderUse docker-container driver
docker buildx use mybuilderSwitch active builder
docker buildx inspect --bootstrapInspect active builder and start it
docker buildx lsList all builders
docker buildx rm mybuilderRemove a builder instance
docker buildx build --platform linux/amd64,linux/arm64 -t user/app:latest --push .Multi-arch build and push to registry
docker buildx build --platform linux/arm/v7 -t user/app:armv7 --load .Build for arm/v7, load into local daemon
docker buildx build --cache-from type=registry,ref=user/app:cache .Use registry as build cache source
docker buildx build --cache-to type=registry,ref=user/app:cache,mode=max .Export cache to registry
docker buildx build --cache-from type=local,src=/tmp/cache --cache-to type=local,dest=/tmp/cache .Local filesystem cache
docker buildx build --output type=oci,dest=./image.tar .Export as OCI tar archive
docker buildx build --output type=local,dest=./out .Export build output to local directory
docker buildx bake -f docker-bake.hclBuild using bake file (HCL/JSON/Compose)
docker buildx imagetools inspect myimage:latestInspect multi-arch manifest
docker buildx imagetools create -t user/app:combined user/app:amd64 user/app:arm64Combine single-arch images into manifest
