PromQL Selectors & Matchers

http_requests_totalMatch metric name
http_requests_total{status="200"}Exact equality match label
http_requests_total{status!="500"}Inequality match label
http_requests_total{job=~"api-.*"}Regex match label
http_requests_total{job!~"test-.*"}Negative regex match label
http_requests_total[5m]Range vector selector (5 min)
http_requests_total offset 1hOffset query by 1 hour
http_requests_total @ 1609459200Evaluate at specific unix epoch

PromQL Operators & Aggregations

sum(node_cpu_seconds_total) by (mode)Sum values, grouping by mode
avg(node_load1) without (instance)Average, excluding instance label
rate(http_requests_total[5m]) * 100Multiply query output by scalar
val1 > bool 50Return boolean 0 or 1 for threshold
q1 and q2Logical AND matching labels
q1 or q2Logical OR combination
q1 ignoring(port) group_left q2Many-to-one label matching

PromQL Essential Functions

rate(http_requests_total[5m])Per-second average rate over range
irate(http_requests_total[2m])Instant per-second rate (last 2 pts)
increase(container_cpu_usage[1h])Total increase in monotonic counter
histogram_quantile(0.99, sum(rate(...)))Calculate 99th percentile latency
predict_linear(node_filesystem_free[1h], 86400)Predict value 24 hours from now
label_replace(q, "dst", "$1", "src", "(.*)")Extract & create new label field
absent(up{job="api"})Returns 1 if vector has no elements

Alertmanager & Alert Rules

# Prometheus Alerting Rule
groups:
  - name: hosts
    rules:
    - alert: InstanceDown
      expr: up == 0
      for: 5m
      labels:
        severity: critical
      annotations:
        summary: "Instance {{ $labels.instance }} is down"
amtool check-config alertmanager.ymlValidate alertmanager configuration
amtool silence add alertname=InstanceDownSilence alert via amtool CLI
